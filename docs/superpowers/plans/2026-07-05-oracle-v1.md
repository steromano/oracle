# Oracle v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. This plan is also the contract source for `/ultracode` parallel execution: **file ownership is exclusive per task** and the `Interfaces` blocks are the integration surface — implement the signatures exactly.

**Goal:** Build Oracle, a self-improving LLM forecasting harness — a deterministic Python package (`oracle/`) plus Claude Code orchestration (CLAUDE.md + skill stubs) plus GitHub Actions automation — with the measurement loop (ledger → resolve → score) airtight and fully tested.

**Architecture:** Strict LLM/Python split (§3.1): all math, storage, dates, and scoring live in the pure, tested `src/oracle/` package reached only through the `oracle` CLI; the LLM does judgment via markdown skills. State is git-tracked JSON on disk (append-only ledger). Connectors are coded to documented API shapes and tested offline against recorded fixtures.

**Tech Stack:** Python 3.12 (uv-managed), pydantic v2, click, httpx, numpy, scipy, matplotlib, jinja2, python-dateutil; pytest + hypothesis. No agent frameworks — Claude Code is the agent framework.

## Global Constraints

_Every task's requirements implicitly include these (copied verbatim from spec / build design):_

- **Python 3.12**, pinned via `uv` (`requires-python = ">=3.12"`). System Python is 3.14; do not rely on it.
- **Dependencies are exactly:** pydantic, click, httpx, numpy, scipy, matplotlib, jinja2, python-dateutil, pytest, hypothesis. No LangChain / agent frameworks (§10.3).
- **Datetimes:** tz-aware **UTC** in storage; rendered in **Australia/Sydney** in reports (§10.4).
- **Append-only ledger:** records are never mutated; corrections are new records referencing the old via `supersedes` (§6.1).
- **CLI stamps time:** `committed_at`, `resolved_at`, and all IDs are assigned by the CLI, never supplied by the caller/LLM. The LLM cannot backdate (§5.9).
- **Narrow interface:** the LLM touches state only through `oracle` CLI subcommands — never edits `data/` directly (§3.1).
- **Blinding is enforced in Python, not prompts:** `BlindCandidate` carries no price/community-forecast/comment fields; `data/sealed/` is read only by the CLI (§5.13, §6.6).
- **Small-N honesty:** the scoreboard/PnL renderer prints `insufficient N` for any cell with N < 30; every headline comparison ships a bootstrap CI (§9.4).
- **Secrets** via `.env` + python-dotenv, never committed; connectors degrade gracefully when a key is absent (§6.6, §10.5).
- **Probability floors/ceilings:** committed probabilities clamp to [0.01, 0.99] absent written extraordinary justification (§5.9).
- **Machine-readable exit codes:** CLI commands used by cron return `0` = nothing to do, `10` = human/LLM attention needed (§10.8).
